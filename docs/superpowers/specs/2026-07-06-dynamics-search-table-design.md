# Дизайн: Графики динамики + Поиск/Таблица для карты дизель-лимитов

**Дата:** 2026-07-06
**Статус:** утверждён пользователем
**Подходы:** A1 (спарклайн цены + таймлайн лимитов) + B1 (поиск + переключатель карта/таблица + фильтры)

## Контекст

Проект «Дизель-лимиты» — оперативная карта ограничений и цен на дизельное топливо для грузоперевозчиков по регионам России. Сбор данных через Hermes-агента (cron 02:00 и 11:00 МСК), фронтенд на D3+SVG, Telegram-канал, SQLite-база `restrictions.db`.

Уже реализовано: 5 уровней жёсткости ограничений, карточка региона с лимитами по сетям АЗС, лента изменений, новости, рост цен за неделю, tooltip, zoom/pan, аккордеон-панели.

### Проблема
1. **Нет динамики.** Карточка региона показывает только текущий снапшот. Пользователь не видит, растёт ли цена и как менялись лимиты. Между тем в БД уже есть `prices_history` (история цен) и `restrictions.previous_value` + `source_date` (история лимитов).
2. **Тяжело искать.** Карта из 85 регионов не приспособлена для быстрого поиска конкретного региона или сравнения нескольких. Нет ни строки поиска, ни табличного вида.

## Цели

- Показать динамику цены и историю изменений лимитов прямо в карточке региона.
- Дать быстрый поиск региона, табличный обзор и фильтрацию — без изменения бэкенда (данные уже на клиенте).

## Не-цели (YAGNI)

- Виртуализация таблицы (85 строк — не нужно).
- Отдельная страница аналитики с большими графиками.
- Большой раскрывающийся график цены (явно отклонён пользователем).
- Фильтр по диапазону цен (явно отклонён пользователем).
- Backend для фильтров и поиска — всё считается на клиенте.
- Серверный полнотекстовый поиск — данные уже загружены во `FEATURES`.

## 1. Архитектура данных

### Расширение `data.json`

В каждый объект `region` добавляется поле `price_spark` — агрегированный массив цен за ~30 дней (до 12 точек), достаточный для отрисовки спарклайна при открытии карточки без задержки:

```json
{
  "region": "Московская область",
  "level": 2,
  "price": 68.45,
  "limit_text": "Газпромнефть: 60 л",
  "truck_limits": [...],
  "weekly_change": 1.20,
  "price_spark": [67.10, 67.30, 67.80, 68.00, 68.20, 68.45]
}
```

Размер: ~12 чисел на регион × 85 регионов ≈ +10 КБ — незначительно.

### Lazy-загрузка истории лимитов

Полная история подгружается отдельным запросом при открытии карточки, чтобы не разбухал основной `data.json`. По одному файлу на регион:

```
GET /static/history/Московская%20область.json
```

```json
{
  "region": "Московская область",
  "prices": [
    {"date": "2026-06-07", "price": 67.10},
    {"date": "2026-06-14", "price": 67.30}
  ],
  "limits": [
    {"date": "2026-06-15", "network": "Газпромнефть", "old": "100 л", "new": "60 л"},
    {"date": "2026-06-20", "network": "Лукойл", "old": null, "new": "40 л"}
  ]
}
```

Имя файла — URL-encoded через `encodeURIComponent`.

### Обоснование
- `price_spark` рендерится мгновенно при открытии карточки — критично для ощущения скорости.
- Полная `history/<region>.json` нужна не всегда, поэтому подгружается lazy. История лимитов относительно объёмна (предыдущие значения + даты).
- Разделение позволяет масштабировать историю (90+ дней) без влияния на размер `data.json`.

## 2. Графики динамики в карточке

При клике на регион существующая карточка `.region-card` дополняется двумя блоками.

### Макет

```
┌─────────────────────────────────────┐
│ ███ card-header (цвет уровня)       │
│   ЖЁСТКИЕ ЛИМИТЫ                    │
│   Московская область             ×  │
├─────────────────────────────────────┤
│ ЦЕНА ДТ                             │
│  68.45 руб/л   ↑ +1.20 ₽/л за нед. │
│  ▁▂▂▃▃▄▅▅▆▆▇   ← спарклайн 30 дн   │
│  мин 67.10 · макс 68.50 · сред 67.6 │
├─────────────────────────────────────┤
│ ОГРАНИЧЕНИЯ                         │
│  Газпромнефть: 60 л                 │
├─────────────────────────────────────┤
│ ДЛЯ ГРУЗОВИКОВ  (существующий блок) │
├─────────────────────────────────────┤
│ 📊 ИСТОРИЯ ИЗМЕНЕНИЙ   ← НОВОЕ      │
│  15.06  Газпромнефть  100 л → 60 л  │ ← ужесточение (красный ▼)
│  20.06  Лукойл        — → 40 л       │ ← новое (оранжевый ▼)
│  01.07  Роснефть      40 л → 100 л   │ ← послабление (зелёный ▲)
└─────────────────────────────────────┘
```

### Спарклайн цены
- Чистый SVG `<polyline>` без библиотек. Размер 120×28 px. Нормализация точек в координаты viewBox.
- Цвет линии = тренд за период:
  - растёт → `#EF4444` (красный)
  - падает → `#10B981` (зелёный)
  - ровно → `#64748B` (серый)
- Под спарклайном — мини-статистика: min / max / среднее за период.
- Рендерится сразу из `price_spark` (данные в `data.json`).

### Таймлайн лимитов
- Список изменений, отсортированный от новых к старым.
- Каждая запись: дата · сеть · `старое → новое` со стрелкой и цветом:
  - **ужесточение** (новое числовое значение меньше старого) — красный ▼
  - **посабление** (новое значение больше старого) — зелёный ▲
  - **новое ограничение** (`old` = null) — оранжевый ▼
  - **снятие ограничения** (`new` = без ограничений) — зелёный ▲
- Парсинг «старое → новое» на основе чисел в `limit_value` (переиспользуем логику `calc_level`).
- Подгружается `fetch('/static/history/' + encodeURIComponent(region) + '.json')` при открытии карточки.
- Состояния: лоадер «загрузка истории…», graceful fallback при ошибке (текст «история недоступна»), пустое состояние «изменений не было».

### Поведение
- Все новые элементы рендерятся внутри существующего `#region-card`.
- XSS-защита через существующую `esc()` обёртку.
- Поведение карточки (открытие/закрытие, Esc) не меняется.

## 3. Поиск + переключатель Карта/Таблица + фильтры

### Расположение

В `header` добавляется строка управления под существующим заголовком и статистикой:

```
┌───────────────────────────────────────────────────────────────┐
│ Дизель-лимиты — оперативная карта для перевозчиков            │
│ Данные: 06.07.2026 · Средняя цена: 67.8 руб/л · 85 регионов   │
├───────────────────────────────────────────────────────────────┤
│ [🔍 Поиск региона...____________]  [🏢 Таблица] [🗺️ Карта]    │ ← НОВОЕ
│ Фильтр: [Все уровни ▾] [Все сети ▾]            [Сбросить]     │ ← НОВОЕ
└───────────────────────────────────────────────────────────────┘
```

### Поиск региона
- Поле ввода с автодополнением (datalist или кастомный dropdown до 8 совпадений).
- При выборе региона:
  - вид «Карта» → зум к региону + открытие карточки
  - вид «Таблица» → подсветка строки + скролл к ней
- Нечёткий поиск: совпадение по подстроке без учёта регистра («татар» → «Республика Татарстан»).

### Переключатель Карта / Таблица
- Toggle-группа из двух кнопок в правой части строки управления.
- **Карта** (по умолчанию): текущий `#map-wrap` без изменений.
- **Таблица** (новый `<div id="table-view">` внутри `#map-wrap`, показывается вместо карты):
  - Колонки: Регион · Уровень · Цена · Лимит · Грузовикам.
  - 85 строк, без виртуализации.
  - Заголовки кликабельны → сортировка asc/desc с индикатором ▲▼.
  - Клик по строке → открывает ту же `.region-card`.
  - Полоса уровня в колонке «Уровень» окрашена цветом из `LEVEL_BY_ID`.

```
Регион              │ Уровень      │ Цена   │ Лимит           │ Грузовикам
Московская область  │ ▲ Жёсткие    │ 68.45  │ ГПН: 60 л       │ Лукойл: 40 л
Республика Татарстан│ ○ Мягкие     │ 65.20  │ —               │ без лимита
Чукотский АО        │ ? Нет данных │  —     │ Нет данных      │ нет данных
```

### Фильтры
Применяются одновременно к карте и таблице:
- **Уровень:** чипсы «Все / Свободно / Мягкие / Средние / Жёсткие / Дефицит / Нет данных» (multi-select кликом).
- **Сеть АЗС:** выпадающий список сетей, собранных из `truck_limits` всех регионов (Газпромнефть, Лукойл, Роснефть, Татнефть и т.д.).
- На карте отфильтрованные регионы **затемняются** (opacity 0.25) вместо скрытия — видна география покрытия.
- На таблице несоответствующие строки скрываются.
- Кнопка «Сбросить фильтры».

### Состояние в URL
Состояние (вид, фильтры, поиск) сохраняется в URL hash для shareable-ссылок:
```
#table;q=татар;lvl=2,3;net=Лукойл
```
Перерисовка только затронутых элементов, без перезагрузки данных.

### Поведение
- Все данные (`FEATURES`) уже на клиенте — фильтрация, сортировка, поиск реализуются чисто на фронтенде.
- Бэкенд для этих функций не меняется.

## 5. Очередь реализации

Каждый шаг — отдельно проверяемый коммит.

1. **Бэкенд-агрегаты.** Расширить `dump_data_json.py` → `price_spark`. Создать `dump_history.py`. Проверка: `data.json` содержит `price_spark`, каталог `/srv/static/history/` заполнен.
2. **Спарклайн в карточке.** Чистый SVG `<polyline>` + мини-статистика. Проверка: при клике на регион виден график цены.
3. **Таймлайн лимитов.** Lazy `fetch('/static/history/...')` + рендер списка изменений. Проверка: история подгружается и красится по типу изменения.
4. **Поиск региона.** Поле с автодополнением + зум/открытие карточки. Проверка: «татар» → Татарстан.
5. **Переключатель Карта/Таблица + рендер таблицы.** Сортировка по колонкам. Проверка: клик по строке открывает карточку.
6. **Фильтры (уровень + сеть) + синхронизация с URL hash.** Затемнение на карте, скрытие в таблице. Проверка: `#table;lvl=3` открывает таблицу с жёсткими лимитами.
7. **Документация** (`CRONS.md`, `README.md`).

## 6. Тестирование и a11y

### Ручное тестирование
- Открыть карточки нескольких регионов, проверить спарклайн и таймлайн.
- Переключить карта/таблица, отсортировать, отфильтровать, поделиться ссылкой.
- Проверка регресса: карта, zoom/pan, tooltip, лента изменений, новости — работают как прежде.

### Доступность
- Таблица с `<thead>` и `scope` на заголовках колонок.
- Фильтры и кнопки доступны с клавиатуры (`:focus-visible` уже стилизован в проекте).
- `aria-live="polite"` на результатах поиска и количестве отфильтрованных строк.
- Состояние toggle-группы отражается через `aria-pressed`.
- Сохраняется существующая поддержка `prefers-reduced-motion` и `Esc` для закрытия карточки.
